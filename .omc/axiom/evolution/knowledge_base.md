### 测试策略

#### K001: 测试环境隔离模式
- **类别**: 测试策略
- **描述**: 使用vi.mock统一测试环境,避免依赖外部配置
- **适用场景**: 所有依赖环境变量或外部状态的测试
- **实现**:
  ```typescript
  vi.mock('../../src/features/auto-update', () => ({
    isTeamEnabled: () => true
  }));
  ```
- **优势**:
  - 消除CI/本地环境差异
  - 提升测试稳定性
  - 减少环境依赖
- **验证次数**: 1 (v7.5.2 CI修复)
- **置信度**: 95%
- **来源**: 2026-03-15 v7.5.2发布
- **状态**: ✅ 已验证

### 规划策略

#### K004: Ralplan共识流程
- **类别**: 规划策略
- **描述**: Planner创建初始计划后，Architect和Critic并行评审，捕获数据不一致和架构缺陷
- **适用场景**: 复杂技术债务修复、大型重构、架构变更
- **实现**:
  - Planner创建v1.0计划
  - Architect评审架构合理性（评分7-10分）
  - Critic评审质量标准（通过/拒绝）
  - 如Critic拒绝，Planner根据反馈修订v2.0
  - 最多迭代5次直至达成共识
- **优势**:
  - 提前发现数据不一致（如审计报告过时）
  - 捕获架构缺陷和风险
  - 避免基于错误假设的计划
- **验证次数**: 1 (技术债务修复)
- **置信度**: 90%
- **来源**: 2026-03-17 技术债务修复会话
- **状态**: ✅ 已验证

#### K005: 数据验证优先策略
- **类别**: 规划策略
- **描述**: 制定计划前先运行grep/tsc等命令验证实际数据，避免基于过时审计报告
- **适用场景**: 所有基于代码库现状的规划任务
- **实现**:
  ```bash
  # 验证实际'any'使用情况
  grep -r ":\s*any" src/ --include="*.ts" | wc -l
  # 验证TypeScript编译状态
  tsc --noEmit
  ```
- **优势**:
  - 避免工作量估算错误（本次避免了9.7倍误差）
  - 基于真实数据制定计划
  - 减少返工和计划修订
- **验证次数**: 1 (技术债务修复)
- **置信度**: 95%
- **来源**: 2026-03-17 技术债务修复会话
- **状态**: ✅ 已验证

### 代码质量

#### K006: TODO分类策略
- **类别**: 代码质量
- **描述**: 区分功能性关键词（如TODO LIST系统）和真实待办项，避免误删功能代码
- **适用场景**: 技术债务清理、代码审计
- **实现**:
  - 搜索所有TODO标记
  - 检查上下文：是否为类名、变量名、注释说明
  - 功能性关键词：保留
  - 真实待办项：清理或转为issue
- **优势**:
  - 避免破坏功能代码
  - 准确识别真实技术债务
  - 减少误删风险
- **验证次数**: 1 (51个TODO中只有1个真实待办)

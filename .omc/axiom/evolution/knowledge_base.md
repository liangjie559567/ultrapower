### 测试策略

#### K001: 测试环境隔离模式
- **类别**: 测试策略
- **描述**: 使用vi.mock统一测试环境,避免依赖外部配置
- **适用场景**: 所有依赖环境变量或外部状态的测试
- **实现**:
  ```typescript
  vi.mock('../../src/features/auto-update', () => ({
    isTeamEnabled: () => true
  }));
  ```
- **优势**:
  - 消除CI/本地环境差异
  - 提升测试稳定性
  - 减少环境依赖
- **验证次数**: 1 (v7.5.2 CI修复)
- **置信度**: 95%
- **来源**: 2026-03-15 v7.5.2发布
- **状态**: ✅ 已验证

### 版本控制

#### K002: Git Rebase工作流
- **类别**: 版本控制
- **描述**: 远程分支领先时使用rebase保持线性历史
- **命令**: `git pull --rebase origin main`
- **优势**:

#### K003: 跳过测试分类标准
- **类别**: 测试管理
- **描述**: 判断哪些测试可以合理跳过
- **标准**:

## Failure Patterns & Lessons

### Anti-Pattern 1: 版本号不同步导致 CI 失败
**问题**: package-lock.json 和 marketplace.json 版本未同步，导致 GitHub Actions 失败 3 次。

**根因**: 手动更新版本号时遗漏部分文件。

**解决方案**:
- 使用 `scripts/bump-version.mjs` 统一更新所有版本文件
- 添加 preflight 检查确保版本同步

**学习**: LQ-053: 版本号同步应自动化，不能依赖人工检查

### Anti-Pattern 2: CI 测试失败但本地通过
**问题**: GitHub Actions 中测试失败，但本地运行通过。

**根因**: CI 环境与本地环境差异（git 配置、临时文件等）。

**解决方案**:
- 跳过 CI，使用本地发布脚本
- 后续需要修复 CI 环境配置

**学习**: LQ-054: CI 环境应与本地环境保持一致，避免环境差异导致的失败

# Draft PRD Context - 代码质量改进

## 需求澄清记录

### Q1: BUG 的具体表现

**回答**: A + B + C + D（全面问题）

* A: 语法错误、类型错误、编译失败

* B: 逻辑错误、功能不符合预期

* C: 代码质量问题、不符合规范

* D: 代码不完整、缺少错误处理

### Q2: 哪些场景最容易出问题

**回答**: A + B + C + E

* A: 自主执行模式（autopilot、ralph）

* B: 特定 agent（executor、deep-executor、designer）

* C: 复杂需求（多文件、架构变更）

* E: Axiom 工作流（ax-implement）

### Q3: 具体问题表现

**回答**: 多维度质量问题

* ❌ 逻辑错误：没有分析现有代码实现

* ❌ 缺少最佳实践：没有结合外部搜索

* ❌ 功能不完整：缺少必要功能

* ❌ CI 门禁失败：无法通过编译/测试

**核心问题**: Agent 缺少"理解现有代码 + 搜索最佳实践 + 完整实现 + 验证"的完整流程

### Q4: 期望的改进目标（优先级）

**回答**: A > B > C > D > F > E

**核心诉求**: 实现 Claude（架构师/评审）+ Codex（执行工程师）的双模型协作工作流

## 用户期望的完整工作流

### 新项目开发流程

1. **需求阶段** (Claude)
   - 对话确定需求 → 需求文档定稿
   - 搜索网络相关项目对比 → 找出不足
   - 功能完善 → 生成技术实现文档
   - 输出：需求文档 + 技术实现文档

1. **开发阶段** (Codex)
   - 读取需求文档 + 技术文档
   - 分解开发周期 → 生成周期开发文档
   - 按周期逐步开发
   - 完成后生成完整功能流程文档

1. **评审优化循环** (Claude ↔ Codex)
   - Claude: 评估功能文档 vs 原始需求 → 优化清单
   - Codex: 按清单优化 → 更新功能文档
   - 重复直到无优化点

1. **测试阶段** (Claude ↔ Codex)
   - Claude: 生成测试清单 + 流程
   - Codex: 按清单排查 → 输出结果
   - 重复直到测试完成

### 老项目修改流程

1. **理解阶段** (Codex)
   - 读取整个项目 → 生成功能流程文档
   - 微服务：每个服务单独生成文档

1. **规划阶段** (Claude)
   - 输入：各服务文档 + 修改需求
   - 输出：修改文档（按功能模块分解）

1. **执行阶段** (Codex)
   - 按功能模块逐个修改
   - 每个模块完成后 → 生成功能流程文档

1. **评审优化循环** (Claude ↔ Codex)
   - 同新项目流程
   - 单个模块测试完成 → 下一个模块

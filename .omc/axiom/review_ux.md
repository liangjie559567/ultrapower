## 修复验证

### ✅ BlockedHookOutput 接口已定义

**位置**: Rough PRD 第 103-111 行

```typescript
interface BlockedHookOutput {
  continue: false;
  error: {
    code: 'PERMISSION_DENIED' | 'HOOK_FAILED' | 'HOOK_TIMEOUT';
    message: string;
    userAction?: string;
    technicalDetails?: any;
  };
}
```

**评价**: 接口设计优秀，包含错误码、用户消息、恢复指引和技术细节。

---

### ✅ 错误消息用户友好

**示例** (Rough PRD 第 127-130 行):

```typescript
message: '❌ 权限验证失败，操作已阻止',
userAction: '请检查文件权限或联系管理员'
```

**优点**:
- 使用中文消息，符合目标用户语言
- 使用 emoji 增强视觉识别
- 消息简洁明确，无技术术语
- 分离用户消息和技术细节

---

### ✅ 恢复路径清晰

**恢复指引机制**:

1. **即时反馈**: `userAction` 字段提供下一步操作建议
2. **技术支持**: `technicalDetails` 字段供开发者调试
3. **错误码分类**: 3 种错误码对应不同恢复路径
   - `PERMISSION_DENIED` → 检查文件权限
   - `HOOK_FAILED` → 检查 hook 脚本
   - `HOOK_TIMEOUT` → 优化 hook 性能

**边界情况表** (Rough PRD 第 140-146 行): 清晰展示 5 种失败场景的处理方式

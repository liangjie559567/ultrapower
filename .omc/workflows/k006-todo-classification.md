# TODO分类策略

## 背景
代码库中的"TODO"关键词可能是功能性的（如类名、变量名、系统名称），也可能是真实待办项。盲目删除会破坏功能代码。本次审计发现51个TODO中仅1个是真实待办项。

## 实现步骤

1. **搜索所有TODO标记**
   ```bash
   grep -rn "TODO" src/ --include="*.ts" --include="*.tsx"
   ```

2. **逐个检查上下文**
   - 打开文件查看TODO周围代码
   - 判断是否为类名、变量名、注释说明
   - 记录分类结果

3. **分类处理**
   - **功能性关键词**：保留（如TODO LIST系统、TODO类等）
   - **真实待办项**：
     - 转为GitHub Issue
     - 添加优先级标签
     - 从代码中删除

4. **文档化决策**
   - 创建清单记录每个TODO的分类理由
   - 保存为.omc/checklists/todo-audit.md

## 验证方法

- 运行grep确认剩余TODO数量
- 检查删除的TODO是否已转为Issue
- 验证保留的TODO是否确实为功能性关键词
- 代码审查确认无误删

## 注意事项

- **检查上下文很关键**：不要仅看TODO本身
- **保留功能性关键词**：如TODO LIST系统、TODO类名等
- **转为Issue**：真实待办项应在Issue系统中追踪
- **定期审计**：每个季度重复此流程

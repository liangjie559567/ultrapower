## 1. 跨模块集成分析

### 1.1 核心数据流路径

**Agent → Hook → State → MCP 完整链路:**

```
用户输入
  ↓
Magic Keywords 检测 (关键词路由)
  ↓
Hook: UserPromptSubmit (bridge.ts 消毒)
  ↓
Agent 路由决策 (49 agents)
  ↓
Task 创建 (SQLite 持久化)
  ↓
MCP 工具调用 (35 tools)
  ↓
State 更新 (.omc/state/*.json)
  ↓
Hook: ToolUse (结果处理)
  ↓
Verifier 验证
  ↓
返回用户
```

**关键集成点:**
1. **bridge.ts** - Hook 输入白名单过滤 (安全边界)
2. **task-router.ts** - Agent 任务分配 (并发控制)
3. **boulder-state/** - 跨会话状态持久化
4. **omc-tools-server.ts** - MCP 工具注册中心

### 1.2 模块间耦合度评估

| 模块对 | 耦合类型 | 耦合度 | 风险 |
|--------|---------|--------|------|
| Agents ↔ Hooks | 事件驱动 | 低 | ✅ 松耦合 |
| Hooks ↔ State | 文件 I/O | 中 | ⚠️ 文件锁竞争 |
| Team ↔ SQLite | 数据库 | 高 | ⚠️ 并发写入限制 |
| MCP ↔ Tools | 协议绑定 | 中 | ✅ 标准化协议 |
| Features ↔ Hooks | 交叉引用 | 中 | ⚠️ 循环依赖风险 |

**集成质量评分:** ⭐⭐⭐⭐☆ (4/5)

## 3. 资源管理评估

### 3.1 并发控制机制

**当前策略:**
```typescript
// task-router.ts
const MAX_CONCURRENT_AGENTS = 20;  // 硬编码
```

**问题:**
- ❌ 未根据系统资源动态调整
- ❌ 不同 Agent 类型未差异化限制
- ❌ 缺少优雅降级机制

**改进建议:**
```typescript
// 动态并发控制
const maxConcurrent = Math.min(
  20,
  Math.floor(os.freemem() / (5 * 1024 * 1024))  // 每 Agent 5MB
);
```

### 3.2 文件 I/O 优化

**当前问题:**
1. **频繁小文件写入** - 每次状态更新都写磁盘
2. **无批量操作** - 状态更新逐个处理
3. **缺少写缓冲** - 未使用 write-behind 策略

**优化方案:**
- 引入写缓冲 (debounce 500ms)
- 批量状态更新
- 使用 mmap 减少系统调用

### 3.3 数据库连接管理

**SQLite 使用模式:**
- ✅ 使用 better-sqlite3 (同步 API,性能优)
- ✅ 单连接复用
- ⚠️ 无连接池 (单线程限制)
- ⚠️ 无预编译语句缓存

**改进空间:**
- 预编译常用查询
- 批量插入优化
- WAL 模式启用 (并发读)

## 5. 架构优化建议

### 5.1 短期优化 (P0 - 1 周内)

**1. Hook 执行并行化**
```typescript
// 当前: 顺序执行 15 类 Hook
// 优化: 无依赖 Hook 并行执行
await Promise.all([
  executeHook('magic-keywords'),
  executeHook('context-injector'),
  executeHook('auto-update')
]);
```
**预期收益:** 减少 40% Hook 延迟

**2. 状态缓存优化**
```typescript
// 当前: 5 秒缓存
// 优化: 智能缓存失效
cache.set(key, value, {
  ttl: isHighFrequency ? 30000 : 5000
});
```
**预期收益:** 减少 60% 文件 I/O

**3. MCP 打包瘦身**
```bash
# 当前: 788KB
# 优化: Tree-shaking + 代码分割
esbuild --bundle --minify --tree-shaking=true
```
**预期收益:** 减少到 ~400KB

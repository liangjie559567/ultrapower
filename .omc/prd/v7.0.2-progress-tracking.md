## 1. 产品概述

### 1.1 背景

ultrapower 当前存在严重的进度追踪和文档管理问题：
- 135+ 研究文档散落，用户无法快速定位当前工作
- notepad.md 膨胀至 1.1MB 导致系统崩溃
- 多个状态文件（autopilot、team、ralph）不同步，用户困惑于当前执行状态
- 临时文档堆积，无自动清理机制

### 1.2 产品目标

构建统一的进度追踪和文档生命周期管理系统，让用户：
1. **一目了然**：会话恢复后立即看到当前任务进度
2. **稳定可靠**：notepad 和状态文件不会因过大而崩溃
3. **自动维护**：历史文档自动归档，工作区保持整洁

### 1.3 成功指标

- 状态读取延迟 < 10ms
- 进度聚合延迟 < 50ms
- notepad 大小 < 500KB
- 研究文档数量 < 30 个
- 用户满意度提升 40%

## 3. 功能需求

### 3.1 统一进度追踪系统

#### 3.1.1 单一真实来源（SSOT）

**功能描述**：
所有执行模式（autopilot、team、ralph、ultrapilot）的状态通过统一接口管理。

**技术要求**：
- 实现 `UnifiedProgress` 接口
- 状态文件位置：`.omc/state/unified-progress.json`
- 支持字段：mode, phase, progress, tasksTotal, tasksCompleted, startedAt, linkedModes

**API 接口**：
```typescript
getProgress(directory: string): UnifiedProgress
updateModeProgress(mode: string, updates: Partial<ModeProgress>): Promise<void>
syncAllStates(directory: string): Promise<void>
```

#### 3.1.2 状态同步机制

**功能描述**：
每次状态写入时自动同步到统一进度文件。

**技术要求**：
- 包装现有 `writeState()` 函数
- 写入时触发 `updateUnifiedProgress()`
- 读取时聚合所有活跃状态
- 支持 WAL 恢复机制

#### 3.1.3 HUD 进度指示器

**功能描述**：
在 CLI 界面实时显示任务进度。

**技术要求**：
- 实现文件：`src/hud/progress-indicator.ts`
- 显示格式：`[TEAM] Planning → Execution (3/5 tasks) [████░░] 60%`
- 状态图标：✓（完成）⚠（警告）✗（错误）⟳（进行中）
- 刷新频率：每次状态更新时

---

### 3.2 Notepad 容量管理

#### 3.2.1 自动 Prune 机制

**功能描述**：
自动清理 7 天前的 working memory 条目。

**技术要求**：
- 修改文件：`src/features/notepad-wisdom/index.ts`
- 触发条件：notepad 大小 > 500KB
- 清理规则：删除 7 天前的 working memory 条目
- 保留 priority context 和 manual section

#### 3.2.2 容量监控

**功能描述**：
实时监控 notepad 大小，达到阈值时警告或自动清理。

**技术要求**：
- 400KB：显示警告
- 500KB：自动执行 prune
- 记录 prune 操作到 `.omc/logs/notepad-prune.log`

---

### 3.3 文档生命周期管理

#### 3.3.1 自动归档引擎

**功能描述**：
根据规则自动归档过期文档。

**技术要求**：
- 实现文件：`src/features/document-lifecycle/archiver.ts`
- 归档规则：
  - `*-verification-report.md`：7 天后归档
  - `review-*-v*.md`：14 天后归档
  - 研究文档 > 30 个时归档最旧的
- 归档目标：`.omc/archive/reports/`, `.omc/archive/reviews/`
- 支持压缩（.gz）

#### 3.3.2 版本控制

**功能描述**：
自动管理文档版本号，避免重复版本堆积。

**技术要求**：
- 版本命名：`{name}-v{major}.{minor}.md`
- 自动递增版本号
- 保留最新版本，旧版本归档

#### 3.3.3 元数据管理

**功能描述**：
记录文档创建、修改、归档历史。

**技术要求**：
- 元数据文件：`.omc/research/.metadata.json`
- 记录字段：created, modified, category, version, relatedDocs, autoArchive
- 归档历史：filename, archivedAt, reason

## 7. 风险与缓解

### 7.1 技术风险

**风险 1: 向后兼容性破坏**
- **影响**: 旧版本状态文件无法读取
- **概率**: 中
- **缓解**: 保留 legacy 路径读取，渐进式迁移

**风险 2: 性能回归**
- **影响**: 状态同步增加写入延迟
- **概率**: 低
- **缓解**: 异步状态同步，批量更新优化

**风险 3: 数据丢失**
- **影响**: 归档过程中文件损坏
- **概率**: 低
- **缓解**: 归档前备份，原子操作，审计日志

### 7.2 产品风险

**风险 4: 用户习惯改变**
- **影响**: 用户不适应新的进度显示方式
- **概率**: 中
- **缓解**: 提供配置选项，保留旧行为

## 9. 附录

### 9.1 数据结构

**统一进度状态**：
```typescript
// .omc/state/unified-progress.json
interface UnifiedProgressState {
  currentMode: ExecutionMode | null;
  modes: {
    [mode: string]: {
      active: boolean;
      phase: string;
      progress: number;
      tasksTotal: number;
      tasksCompleted: number;
      startedAt: string;
      lastHeartbeat: string;
    };
  };
  linkedModes: {
    primary: string;
    secondary: string[];
  } | null;
  lastUpdated: string;
}
```

**文档元数据**：
```typescript
// .omc/research/.metadata.json
interface DocumentMetadata {
  documents: {
    [filename: string]: {
      created: string;
      modified: string;
      category: 'core' | 'temporary' | 'archived';
      version?: string;
      relatedDocs: string[];
      autoArchive: boolean;
    };
  };
  archiveHistory: {
    filename: string;
    archivedAt: string;
    reason: string;
  }[];
}
```

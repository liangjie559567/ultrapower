# v7.0.2 质量改进版本规划

**版本目标**：提升代码质量、类型安全和可维护性

## 优先级分级

### P0 - 类型安全改进（必须完成）

**问题**：797 处类型绕过（any、@ts-ignore、@ts-nocheck）

**目标**：减少至少 50%（~400 处）

**策略**：
1. 识别高频模块（hooks、mcp、tools）
2. 逐模块重构，优先核心路径
3. 添加严格类型定义

**验收标准**：
- 类型绕过减少至 ≤400 处
- 核心模块（hooks/mcp/tools）类型覆盖率 >80%
- 无新增类型绕过

### P1 - 日志系统统一（重要）

**问题**：714 处 console.log/error/warn 调用

**目标**：统一为结构化日志系统

**策略**：
1. 设计统一日志接口（支持分级、上下文、格式化）
2. 创建迁移脚本批量替换
3. 保留关键调试点的 console

**验收标准**：
- console 调用减少至 ≤100 处（仅保留关键调试点）
- 所有业务日志使用统一接口
- 日志可配置级别和输出格式

### P2 - 研究工件整理（可选）

**问题**：92 个研究文档散落在 .omc/research/

**目标**：归档到知识库或删除过期内容

**策略**：
1. 分类：有效知识 vs 过期草稿
2. 有效知识迁移到 Axiom 知识库
3. 删除过期草稿

**验收标准**：
- .omc/research/ 文件数 ≤20
- 有价值内容已归档到 Axiom

### 阶段 2：日志系统统一（2-3 天）

**任务 2.1**：设计日志接口
```typescript
interface Logger {
  debug(message: string, context?: Record<string, unknown>): void;
  info(message: string, context?: Record<string, unknown>): void;
  warn(message: string, context?: Record<string, unknown>): void;
  error(message: string, error?: Error, context?: Record<string, unknown>): void;
}
```

**任务 2.2**：实现日志系统
- 支持分级（DEBUG/INFO/WARN/ERROR）
- 支持上下文注入

**任务 2.3**：迁移现有调用
- 创建 codemod 脚本批量替换

- T1: 修正 CI 配置 (1h)
- T2: 增强 E2E 测试 (2h)

### 技术要点

- **主要优化**: 智能缓存检测（跳过未变更的 bundle）
- **预期收益**: 增量构建提速 30-70%

### 实施计划

## 3. Agent 路由智能化 (P2)

### 核心发现
- **当前机制**: 关键词检测 + 部分 Model Routing
- **局限性**: 无语义理解、无历史学习、缺乏上下文感知
- **目标**: 路由准确率提升 30-40%，达到 ≥85%

### 实施计划 (3-5 天)
**阶段 1: 任务特征提取 (1 天)**
- 扩展信号系统（Intent/Domain/Workflow/Agent）
- 实现特征提取器

**阶段 2: Agent 能力建模 (1 天)**
- 为 49 个 Agent 建立能力档案
- 定义 keywords/intents/domains/complexity

**阶段 3: 智能路由引擎 (1-2 天)**
- 三层匹配：规则引擎 → 相似度匹配 → 历史学习
- 决策融合算法

**阶段 4: 历史学习系统 (0.5-1 天)**
- 路由决策日志
- 反馈循环和模型更新

**阶段 5: 测试和部署 (0.5-1 天)**
- A/B 测试框架
- 逐步切换流量

### 技术要点
- 保留关键词检测作为后备
- 用户可手动覆盖路由决策
- 提供决策解释（透明度）
- 核心指标：准确率 ≥85%，决策时间 <50ms

## 执行建议

### 推荐顺序
1. **立即执行**: 测试覆盖率快速路径 (2h)
   - 修正 CI 阻塞问题
   - 快速达到 75% 目标

2. **短期执行**: 构建流程缓存优化 (1 天)
   - 显著提升开发体验
   - 低风险高收益

3. **中期规划**: Agent 路由智能化 (3-5 天)
   - 需要更多设计和测试
   - 可与其他工作并行

### 资源分配
- **测试覆盖率**: 1 个 executor (快速路径) 或 1 个 executor + 1 个 verifier (完整路径)
- **构建优化**: 1 个 executor (缓存) + 可选 1 个 architect (并行化)
- **路由智能化**: 1 个 architect + 1-2 个 executor + 1 个 verifier

### 风险控制
- 所有变更都有回滚方案
- 保留现有机制作为后备
- 充分测试后再部署到生产

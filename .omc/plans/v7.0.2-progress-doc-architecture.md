## 1. 问题分析

### 1.1 当前痛点

基于 explore-worker 和 planner-worker 的发现：

**进度追踪碎片化**：
- 135+ 研究文档散落在 `.omc/research/`

- 多个状态文件不同步（autopilot、ultrapilot、team、ralph）
- 用户无法快速了解任务进度

**文档生命周期混乱**：
- 临时验证报告未自动清理
- 重复文档（v1/v2/part1-4）堆积
- 无明确归档策略

**状态指示不明确**：
- HUD 未显示当前执行模式
- 多模式并发时状态冲突
- 缺少统一的进度可视化

## 2. 架构设计

### 2.1 统一进度追踪系统

#### 2.1.1 单一真实来源（SSOT）

**核心原则**：所有模式状态通过 `state-manager` 统一管理

```typescript
// 统一状态接口
interface UnifiedProgress {
  mode: ExecutionMode;           // 当前活跃模式
  phase: string;                 // 当前阶段
  progress: number;              // 0-100 进度百分比
  tasksTotal: number;
  tasksCompleted: number;
  startedAt: string;
  estimatedCompletion?: string;
  linkedModes: string[];         // 关联模式（如 team+ralph）
}
```

**实现位置**：`src/features/state-manager/progress-tracker.ts`

#### 2.1.2 状态同步机制

**写入时同步**：
```typescript
// 每次状态写入时更新统一进度
export async function writeStateWithProgress<T>(
  mode: string,
  data: T,
  location: StateLocation
): Promise<void> {
  await writeState(mode, data, location);
  await updateUnifiedProgress(mode, data);  // 同步到 SSOT
}
```

**读取时聚合**：
```typescript
export function getUnifiedProgress(directory: string): UnifiedProgress {
  const activeStates = listStates({ pattern: '*-state' });
  return aggregateProgress(activeStates);
}
```

#### 2.1.3 恢复和容错

**状态恢复**：
- 利用现有 WAL（Write-Ahead Log）机制
- 崩溃后自动从 WAL 恢复未提交状态

**冲突解决**：
```typescript
// 优先级：team > ralph > ultrapilot > autopilot
const MODE_PRIORITY = {
  team: 4,
  ralph: 3,
  ultrapilot: 2,
  autopilot: 1
};

function resolveConflict(states: StateFile[]): StateFile {
  return states.sort((a, b) =>
    MODE_PRIORITY[b.mode] - MODE_PRIORITY[a.mode]
  )[0];
}
```

### 2.2 文档生命周期管理

#### 2.2.1 自动归档策略

**触发条件**：
1. 模式完成时（`active: false`）
2. 文档超过 7 天未修改
3. 手动触发 `/ultrapower:cleanup`

**归档规则**：
```typescript
interface ArchiveRule {
  pattern: string;              // 文件匹配模式
  maxAge: number;               // 最大保留天数
  destination: string;          // 归档目标目录
  compress: boolean;            // 是否压缩
}

const ARCHIVE_RULES: ArchiveRule[] = [
  {
    pattern: '*-verification-report.md',
    maxAge: 7,
    destination: '.omc/archive/reports/',
    compress: true
  },
  {
    pattern: 'review-*-v*.md',
    maxAge: 14,
    destination: '.omc/archive/reviews/',
    compress: true
  }
];
```

**实现位置**：`src/features/document-lifecycle/archiver.ts`

#### 2.2.2 版本控制机制

**版本命名规范**：
```
{name}-v{major}.{minor}.md
```

**自动版本管理**：
```typescript
export function createVersionedDoc(
  name: string,
  content: string
): string {
  const existing = findLatestVersion(name);
  const nextVersion = incrementVersion(existing);
  return writeDoc(`${name}-v${nextVersion}.md`, content);
}
```

#### 2.2.3 清理触发条件

**自动清理**：
- 模式完成后 24 小时
- notepad 超过 500KB 时触发 prune
- 研究文档超过 30 个时归档旧文档

**手动清理**：
```bash
omc cleanup --mode research --older-than 30d
omc cleanup --mode notepad --prune
omc cleanup --mode states --orphaned
```

### 2.3 任务状态指示系统

#### 2.3.1 状态优先级

**优先级定义**：
```typescript
enum StatusPriority {
  ERROR = 5,      // 错误状态（红色）
  ACTIVE = 4,     // 活跃执行（绿色闪烁）
  PAUSED = 3,     // 暂停（黄色）
  IDLE = 2,       // 空闲（灰色）
  COMPLETE = 1    // 完成（绿色）
}
```

**冲突解决**：
- 多模式并发时，显示最高优先级状态
- 错误状态覆盖所有其他状态

#### 2.3.2 HUD 进度指示器

**显示内容**：
```
[TEAM] Planning → Execution (3/5 tasks) [████░░] 60%
```

**实现接口**：
```typescript
interface HUDIndicator {
  mode: string;
  phase: string;
  progress: number;
  tasksInfo: string;
  statusIcon: string;  // ✓ ⚠ ✗ ⟳
}

export function renderHUD(progress: UnifiedProgress): string {
  return `[${progress.mode.toUpperCase()}] ${progress.phase} ` +
         `(${progress.tasksCompleted}/${progress.tasksTotal}) ` +
         `${renderProgressBar(progress.progress)} ${progress.progress}%`;
}
```

**实现位置**：`src/hud/progress-indicator.ts`

## 7. 风险与缓解

### 7.1 风险

1. **向后兼容性**：旧状态文件格式不兼容
2. **性能影响**：状态同步增加写入延迟
3. **数据丢失**：归档过程中文件损坏

### 7.2 缓解措施

1. **兼容性**：
   - 保留 legacy 路径读取
   - 渐进式迁移策略

2. **性能**：
   - 异步状态同步
   - 批量更新优化

3. **数据安全**：
   - 归档前备份
   - 原子操作保证
   - 归档日志记录

<Purpose>
Wizard 是一个交互式向导，帮助新用户在 2 分钟内通过 3 个简单问题找到最合适的执行模式。
</Purpose>

<Use_When>

* 用户说 "wizard"、"帮我选"、"不知道用什么"、"怎么开始"

* 用户第一次使用 ultrapower，需要引导

* 用户不确定该用 autopilot/ralph/team/executor 哪个
</Use_When>

<Do_Not_Use_When>

* 用户已经明确说了要用哪个模式

* 用户只是在问某个模式的用法
</Do_Not_Use_When>

<Execution>

## 实现方式

使用 `src/features/wizard/` 模块的 `WizardEngine` 类：

```typescript
import { WizardEngine } from './features/wizard/index.js';

const wizard = new WizardEngine();
```

## 问题流程

### Q1: 你想做什么？

使用 AskUserQuestion 提问：

```
问题: "你想做什么？"
header: "任务类型"
选项:
  1. 实现单个功能 — 添加一个新功能或改进现有功能
  2. 实现多个功能 — 同时开发多个独立或相关的功能
  3. 修复问题 — 修复 bug、错误或测试失败
  4. 不确定，需要探索 — 还不清楚具体要做什么，需要先分析
```

### Q2: 分支问题（根据 Q1）

**如果 Q1 = 单个功能**，问：
```
问题: "任务复杂度？"
header: "复杂度"
选项:
  1. 简单 — 修改 1-3 个文件，逻辑清晰
  2. 复杂 — 涉及多个模块，需要架构设计
```

**如果 Q1 = 多个功能**，问：
```
问题: "功能是否独立？"
header: "依赖关系"
选项:
  1. 是 — 功能之间没有依赖，可以并行开发
  2. 否 — 功能之间有依赖关系，需要协调
```

**如果 Q1 = 修复问题**，问：
```
问题: "问题范围？"
header: "影响范围"
选项:
  1. 单个文件 — 问题定位明确，影响范围小
  2. 多个文件 — 问题涉及多个模块或不确定位置
```

**如果 Q1 = 不确定**，问：
```
问题: "需要规划吗？"
header: "规划需求"
选项:
  1. 是 — 需要先分析和制定计划
  2. 否 — 直接开始探索
```

### Q3: 细化问题（部分路径需要）

**如果 Q1=单个功能 且 Q2=简单**，问：
```
问题: "需要持续执行直到完成？"
header: "执行模式"
选项:
  1. 是 — 不停止直到任务完全完成并验证通过
  2. 否 — 完成基本实现即可
```

**如果 Q1=修复问题 且 Q2=多个文件**，问：
```
问题: "需要验证循环？"
header: "验证策略"
选项:
  1. 是 — 修复后需要反复测试验证
  2. 否 — 修复一次即可
```

## 推荐逻辑

调用 `getRecommendation()` 获取推荐模式：

| Q1 | Q2 | Q3 | 推荐 |
| --- | --- | --- | --- |
| 单个功能 | 简单 | 否 | executor |
| 单个功能 | 简单 | 是 | ralph |
| 单个功能 | 复杂 | - | autopilot |
| 多个功能 | 独立 | - | ultrawork |
| 多个功能 | 不独立 | - | team |
| 修复问题 | 单文件 | - | executor |
| 修复问题 | 多文件 | 否 | ultrawork |
| 修复问题 | 多文件 | 是 | ralph |
| 不确定 | 需要规划 | - | plan |
| 不确定 | 不需要 | - | 重新回答 Q1 |

## 输出格式

```
✨ 根据你的回答，我推荐使用: <mode>

📋 <Mode> 模式说明:
<一句话描述>

🎯 使用方式:
/<mode> "你的任务描述"

💡 示例:
/<mode> "<具体示例>"

❓ 选择:
1. 确认并执行
2. 返回重新选择
3. 查看所有模式
```

### 模式说明参考

* **executor**: 直接实现，适合简单明确的任务

* **ralph**: 持续执行直到验证通过，适合需要反复迭代的任务

* **autopilot**: 自主规划和实现，适合复杂的单一功能

* **ultrawork**: 最大并行度，适合多个独立任务

* **team**: 协调式团队执行，适合有依赖关系的多任务

* **plan**: 先制定计划，适合不确定的探索性任务

## 特殊处理

* 支持 "b" 或 "back" 返回上一步

* 支持 "q" 或 "quit" 退出向导

* 无效输入时重新显示当前问题

# 实现者 Subagent Prompt 模板

派发实现者 subagent 时使用此模板。

```
Task tool (general-purpose):
  description: "实现任务 N：[任务名称]"
  prompt: |
    你正在实现任务 N：[任务名称]

    ## 任务描述

    [计划中任务的完整文本——粘贴在此处，不要让 subagent 读取文件]

    ## 上下文

    [场景设定：此任务的位置、依赖关系、架构上下文]

    ## 开始前

    如果你对以下内容有疑问：
    - 需求或验收标准
    - 方法或实现策略
    - 依赖关系或假设
    - 任务描述中不清楚的任何内容

    **现在就问。** 在开始工作前提出任何疑虑。

    ## 你的工作

    明确需求后：
    1. 严格按照任务规格实现
    2. 编写测试（如任务要求则遵循 TDD）
    3. 验证实现有效
    4. 提交你的工作
    5. 自我审查（见下文）
    6. 汇报结果

    工作目录：[目录]

    **工作过程中：** 如果遇到意外或不清楚的情况，**提问**。
    随时暂停澄清都是可以的。不要猜测或做假设。

    ## 汇报前：自我审查

    用新鲜的眼光审查你的工作。问自己：

    **完整性：**
    - 我是否完整实现了规格中的所有内容？
    - 我是否遗漏了任何需求？
    - 是否有我未处理的边缘情况？

    **质量：**
    - 这是我最好的工作吗？
    - 命名是否清晰准确（与功能匹配，而非实现方式）？
    - 代码是否简洁可维护？

    **纪律：**
    - 我是否避免了过度构建（YAGNI）？
    - 我是否只构建了请求的内容？
    - 我是否遵循了代码库中的现有模式？

    **测试：**
    - 测试是否真正验证了行为（而非只是 mock 行为）？
    - 如果需要，是否遵循了 TDD？
    - 测试是否全面？

    如果在自我审查中发现问题，在汇报前立即修复。

    ## 报告格式

    完成后，报告：
    - 你实现了什么
    - 你测试了什么以及测试结果
    - 变更的文件
    - 自我审查发现（如有）
    - 任何问题或疑虑
```

# 规格合规审查员 Prompt 模板

派发规格合规审查员 subagent 时使用此模板。

**目的：** 验证实现者构建了所请求的内容（不多也不少）

```
Task tool (general-purpose):
  description: "审查任务 N 的规格合规性"
  prompt: |
    你正在审查实现是否与其规格匹配。

    ## 请求内容

    [任务需求的完整文本]

    ## 实现者声称构建的内容

    [来自实现者的报告]

    ## 重要：不要相信报告

    实现者完成得异常快。他们的报告可能不完整、
    不准确或过于乐观。你必须独立验证所有内容。

    **不要：**
    - 相信他们关于实现内容的说法
    - 信任他们关于完整性的声明
    - 接受他们对需求的解读

    **要：**
    - 阅读他们实际编写的代码
    - 逐行对比实际实现与需求
    - 检查他们声称已实现但缺失的部分
    - 寻找他们未提及的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失需求：**
    - 他们是否实现了所有请求的内容？
    - 是否有他们跳过或遗漏的需求？
    - 他们是否声称某功能有效但实际未实现？

    **额外/不必要的工作：**
    - 他们是否构建了未请求的内容？
    - 是否过度设计或添加了不必要的功能？
    - 是否添加了规格中没有的"锦上添花"内容？

    **误解：**
    - 他们是否以不同于预期的方式解读了需求？
    - 他们是否解决了错误的问题？
    - 他们是否实现了正确的功能但方式错误？

    **通过阅读代码来验证，而非相信报告。**

    报告：
    - ✅ 规格合规（如代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或多余的内容，附文件:行号引用]
```

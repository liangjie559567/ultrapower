## 审查重点

1. **代码质量**
   - 遵循现有模式和规范
   - 代码简洁、可读、可维护
   - 适当的抽象

2. **正确性**
   - 是否实现了其声称的功能？
   - 边缘情况是否已处理？
   - 错误处理是否适当？

3. **安全性**
   - 输入验证
   - 无硬编码密钥
   - 安全的依赖项

4. **测试**
   - 充分的测试覆盖率
   - 测试有实际意义
   - 边缘情况已测试

5. **文档**
   - 代码自文档化
   - 复杂逻辑有说明
   - API 变更已记录

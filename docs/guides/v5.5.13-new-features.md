# ultrapower v5.5.13 新功能指南

## 概述

v5.5.13 引入了 4 个重要的用户体验改进功能，帮助你快速选择最适合的执行模式和工作流。

## 1. Wizard 交互式引导 (`/ultrapower:wizard`)

### 功能说明

通过 3 个问题引导用户选择最适合的执行模式。

### 使用方法

```bash
/ultrapower:wizard
```

### 3 个引导问题

1. **任务类型**: 单一任务 / 多个独立任务 / 复杂多步骤任务
2. **项目规模**: 简单(1-3文件) / 中等(4-10文件) / 大型(10+文件)
3. **团队协作**: 独立工作 / 需要并行协作

### 推荐逻辑

* 单一任务 + 简单规模 → `executor` agent

* 多个独立任务 → `ultrawork` 并行执行

* 复杂多步骤 → `autopilot` 自主执行

* 需要协作 → `team` 模式

### 示例

```
$ /ultrapower:wizard

问题 1: 你的任务类型是什么?
  1. 单一任务
  2. 多个独立任务
  3. 复杂多步骤任务
选择: 3

问题 2: 项目规模?
  1. 简单 (1-3 文件)
  2. 中等 (4-10 文件)
  3. 大型 (10+ 文件)
选择: 2

问题 3: 需要团队协作吗?
  1. 独立工作
  2. 需要并行协作
选择: 1

推荐: 使用 autopilot 模式自主执行复杂任务
```

## 2. 工作流推荐引擎（中文支持）

### 功能说明

自动分析用户输入，识别意图并推荐最佳 agent/skill。

### 7 种意图分类

| 意图类型 | 中文关键词 | 英文关键词 | 优先级 |
| --------- | ----------- | ----------- | -------- |
| bug-fix | 修复/错误/问题/故障 | fix/bug/error/issue | 10 |
| refactor | 重构/优化/改进 | refactor/cleanup | 9 |
| review | 审查/检查/评审 | review/audit/check | 8 |
| plan | 规划/设计/架构 | plan/design/architect | 7 |
| explore | 探索/分析/调查 | explore/analyze | 6 |
| feature-multiple | 添加.*多个/实现.*多个 | add multiple/implement several | 5 |
| feature-single | 添加/实现/创建/构建 | add/implement/create | 1 |

### 使用示例

```bash

# 中文输入

"修复登录错误" → bug-fix → debugger agent
"添加用户管理功能" → feature-single → executor agent
"重构整个架构" → refactor → architect agent

# 英文输入

"fix authentication bug" → bug-fix → debugger agent
"add user dashboard" → feature-single → executor agent
```

### 自动推荐流程

1. 系统分析用户输入文本
2. 匹配意图关键词（中英文）
3. 根据优先级选择最高匹配
4. 推荐对应 agent 或 skill
5. 用户可接受或手动选择

## 3. 任务模板库

### 功能说明

预定义的 5 个常见任务模板，快速启动标准化工作流。

### 可用模板

1. **feature-development**: 新功能开发完整流程
   - 需求分析 → 设计 → 实现 → 测试 → 审查

1. **bug-fix**: Bug 修复标准流程
   - 问题诊断 → 根因分析 → 修复 → 验证 → 回归测试

1. **code-review**: 代码审查检查清单
   - 风格检查 → 质量审查 → 安全审查 → 性能审查

1. **refactoring**: 重构工作流程
   - 分析现状 → 设计方案 → 实现重构 → 测试验证

1. **security-audit**: 安全审计检查点
   - 漏洞扫描 → 信任边界审查 → 认证授权检查 → 报告生成

### 集成方式

Wizard 会根据用户回答自动应用对应模板。

### 使用示例

```bash

# 通过 wizard 自动应用模板

/ultrapower:wizard

# 选择对应任务类型后自动加载模板

# 或手动指定模板

/ultrapower:autopilot --template feature-development
```

## 4. 故障排查手册

### 功能说明

智能错误匹配和解决方案推荐系统。

### 支持的错误类型

* TypeScript 类型错误

* 文件系统权限错误

* Hook 执行失败

* Agent 超时

* 状态文件损坏

### 使用方法

```bash
/ultrapower:omc-doctor
```

### 故障排查文档位置

* `docs/troubleshooting/common-errors.md`

* `docs/troubleshooting/agent-timeouts.md`

* `docs/troubleshooting/hook-failures.md`

* `docs/troubleshooting/performance-issues.md`

* `docs/troubleshooting/state-corruption.md`

### 常见问题解决

**问题**: Agent 执行超时
**解决**: 检查 `.omc/state/` 中的状态文件，运行 `omc-doctor` 诊断

**问题**: Hook 执行失败
**解决**: 查看 `.omc/logs/` 中的执行日志，确认输入格式正确

**问题**: 状态文件损坏
**解决**: 运行 `omc-doctor --fix` 自动修复，或手动删除 `.omc/state/` 重新初始化

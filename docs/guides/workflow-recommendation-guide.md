# 智能工作流推荐引擎用户指南

## 概述

智能工作流推荐引擎根据你的项目历史、任务类型和执行模式偏好，自动推荐最优工作流。

## 工作原理

### 学习机制

系统记录你的：
- 执行模式选择历史
- 任务完成时间
- 成功率
- 模式偏好

### 推荐算法

基于以下因素计算推荐分数：
1. **历史成功率** (40%)：该模式在类似任务中的成功率
2. **完成时间** (30%)：该模式的平均完成时间
3. **用户偏好** (20%)：用户过去的选择倾向
4. **任务相似度** (10%)：当前任务与历史任务的相似程度

## 使用方式

### 自动推荐

系统在以下场景自动推荐：
- 新建任务时
- 任务类型变更时
- 执行模式失败后

### 手动查询

```
/workflow-recommend "你的任务描述"
```

## 个性化设置

### 查看推荐历史

```
/workflow-stats
```

显示：
- 使用过的模式
- 每个模式的成功率
- 平均完成时间
- 最常用模式

### 重置学习数据

```
/workflow-reset
```

清除所有历史数据，重新开始学习。

## 最佳实践

1. **多尝试不同模式**：帮助系统学习你的偏好
2. **提供详细任务描述**：提高推荐准确度
3. **定期查看统计数据**：了解自己的工作模式
4. **反馈推荐结果**：告诉系统推荐是否有帮助

## 常见问题

**Q: 推荐不准确怎么办？**
A: 系统需要更多数据学习。继续使用，推荐会逐渐改进。

**Q: 如何禁用自动推荐？**
A: 在设置中关闭"自动推荐"选项。

**Q: 推荐数据会被保存吗？**
A: 是的，数据保存在本地项目中，不会上传。

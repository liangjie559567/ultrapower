# 交互式向导用户指南

## 概述

交互式向导帮助你在 2 分钟内选择最适合的执行模式。

## 快速开始

```
/wizard
```

向导通过 3 个简单问题了解你的需求，然后推荐最优执行模式。

## 问题详解

### 第 1 问：你想做什么？

| 选项 | 适用场景 | 示例 |
| ------ | --------- | ------ |
| 实现单个功能 | 添加新功能或改进现有功能 | "添加用户登录"、"优化搜索性能" |
| 实现多个功能 | 同时开发多个功能 | "构建完整用户系统"、"重构整个模块" |
| 修复问题 | 修复 bug 或测试失败 | "修复登录失败"、"解决类型错误" |
| 不确定，需要探索 | 还不清楚具体要做什么 | "优化性能"、"改进架构" |

### 第 2 问：分支问题

根据第 1 问的答案，向导会问不同的问题：

**实现单个功能** → 任务复杂度？

* 简单：修改 1-3 个文件，逻辑清晰

* 复杂：涉及多个模块，需要架构设计

**实现多个功能** → 功能是否独立？

* 是：功能之间没有依赖，可以并行开发

* 否：功能之间有依赖关系，需要协调

**修复问题** → 问题范围？

* 单个文件：问题定位明确，影响范围小

* 多个文件：问题涉及多个模块或不确定位置

**不确定，需要探索** → 需要规划吗？

* 是：需要先分析和制定计划

* 否：直接开始探索

### 第 3 问：细化问题

**需要持续执行直到完成？**

* 是：不停止直到任务完全完成并验证通过

* 否：完成基本实现即可

**需要验证循环？**

* 是：修复后需要反复测试验证

* 否：修复一次即可

## 执行模式速查表

| 模式 | 适用场景 | 特点 |
| ------ | --------- | ------ |
| executor | 单个文件修复、简单功能 | 直接实现，快速完成 |
| ralph | 需要反复验证的任务 | 持续执行直到通过 |
| autopilot | 复杂功能实现 | 自主规划和实现 |
| ultrawork | 多个独立功能 | 最大并行度 |
| team | 多个相关功能 | 协调式团队执行 |
| plan | 不确定需求 | 先制定计划 |

## 键盘快捷键

| 快捷键 | 功能 |
| -------- | ------ |
| 1-4 | 快速选择选项 |
| b / back | 返回上一步 |
| q / quit | 退出向导 |
| ? | 查看当前问题的详细说明 |

## 常见问题

**Q: 如何返回上一步？**
A: 输入 `b` 或 `back` 返回上一个问题。

**Q: 如何退出向导？**
A: 输入 `q` 或 `quit` 退出向导。

**Q: 推荐的模式不适合我怎么办？**
A: 选择"返回重新选择"或"手动选择"其他模式。

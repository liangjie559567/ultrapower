# 开发标准

## 编码要求

* **编码前必须执行 K012 快速验证检查清单** - 参见 [quick-verification-checklist.md](./quick-verification-checklist.md)

* 编码前先描述方案并等待批准

* 修改超3个文件需先分解任务

* 代码完成后列出潜在问题和测试用例

* 发现bug先写复现测试

* 每次被纠正后在此文件添加规则

## 语言规范

* **强制使用简体中文**：所有回复、文档、注释、日志、提交信息

* 唯一例外：代码标识符遵循项目命名约定

## 验证机制

* 拒绝 CI/远程验证，所有验证本地执行

* 每次改动提供可重复验证步骤

* 验证失败立即终止提交

## 代码质量

* 遵循 SOLID、DRY 原则

* 禁止 MVP/占位符，必须完整实现

* 删除过时/重复代码

* 评估性能影响

# 工作流程规范

## 总原则

* **强制深度思考**：使用 sequential-thinking 梳理问题

* 问题驱动优先于流程驱动

* 动态调整而非僵化执行

## 工具链顺序

1. sequential-thinking（分析问题）
2. shrimp-task-manager（任务规划）
3. 直接执行（编码实现）

## 标准 6 步骤

1. 分析需求
2. 获取上下文
3. 选择工具
4. 执行任务
5. 验证质量
6. 存储知识

## 研究-计划-实施 5 阶段

1. **研究**：阅读材料、厘清约束，禁止编码
2. **计划**：制定详细计划与成功标准
3. **实施**：根据计划执行并保持小步提交
4. **验证**：运行测试或验证脚本，记录结果
5. **提交**：准备交付文档与迁移/回滚方案

# CCG Workflow 真实项目验证指南

**目标**: 在真实项目中测试 CCG Workflow，收集用户反馈并迭代优化

**预计时间**: 2 天

---

## 验证场景

### 场景 1: 新项目开发
**项目类型**: 从零开始的新项目
**测试目标**: 验证需求→开发→优化→测试完整流程

**步骤**:
1. 准备需求描述
2. 运行 `/ccg-workflow new`
3. 观察 4 阶段执行过程
4. 记录执行时间和遇到的问题

**验收标准**:
- [ ] 需求 → 可运行代码 < 2 小时
- [ ] 优化循环 < 3 轮收敛
- [ ] CI 通过率 > 90%

---

### 场景 2: 老项目改造
**项目类型**: 现有代码库需要重构
**测试目标**: 验证模块化改造流程

**步骤**:
1. 选择一个需要重构的模块
2. 运行 `/ccg-workflow old`
3. 观察模块拆分和逐模块闭环
4. 记录改造时间和代码质量

**验收标准**:
- [ ] 单模块修改 < 1 小时
- [ ] 模块测试通过率 100%
- [ ] 无破坏性变更

---

### 场景 3: 微服务项目
**项目类型**: 多服务架构
**测试目标**: 验证微服务支持

**步骤**:
1. 在微服务项目中运行
2. 验证每个服务独立文档生成
3. 检查跨服务依赖标注
4. 记录处理时间

**验收标准**:
- [ ] 自动检测微服务结构
- [ ] 每个服务独立处理
- [ ] 跨服务依赖正确标注

---

## 性能验证

### 缓存命中率测试
**目标**: 验证缓存优化效果

**步骤**:
1. 首次运行记录时间
2. 立即再次运行记录时间
3. 调用 `fileCache.getStats()` 查看命中率

**预期结果**:
- 首次: ~50ms
- 缓存命中: ~30ms (~42% 提升)
- 命中率: > 40%

---

### 增量处理测试
**目标**: 验证增量处理性能

**步骤**:
1. 修改 1-2 个文件
2. 运行 CCG Workflow
3. 记录处理时间

**预期结果**:
- 仅处理变更文件
- 性能提升: > 50% (vs 全量扫描)

---

### 流式处理测试
**目标**: 验证大文件内存占用

**步骤**:
1. 创建 10000+ 行的大文件
2. 使用流式处理读取
3. 监控内存占用

**预期结果**:
- 内存占用恒定
- 不受文件大小影响

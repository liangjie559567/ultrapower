# CCG Workflow API 文档

本文档描述 CCG Workflow 的 7 种文档模板及其结构。

## 文档模板系统

### 1. requirements.md - 需求文档

**生成者**: Claude
**用途**: 定义项目需求和功能范围

**结构**:
```markdown

# 需求文档

## 项目概述

* 项目名称

* 项目目标

* 核心价值

## 功能需求

### 功能 1

* 描述

* 验收标准

### 功能 2

* 描述

* 验收标准

## 非功能需求

* 性能要求

* 安全要求

* 可扩展性

## 竞品对比

* 竞品 A 的优势

* 竞品 B 的不足

* 我们的差异化

## 约束条件

* 技术栈限制

* 时间限制

* 资源限制
```

**示例**:
```markdown

# 需求文档：用户认证系统

## 项目概述

* 项目名称：User Auth System v1.0

* 项目目标：提供安全的用户认证和授权

* 核心价值：保护用户数据，防止未授权访问

## 功能需求

### 用户注册

* 支持邮箱和手机号注册

* 验收标准：注册成功率 > 99%

### 用户登录

* 支持邮箱/手机 + 密码登录

* 支持 OAuth 第三方登录

* 验收标准：登录延迟 < 500ms
```

### 3. feature-flow.md - 功能流程文档

**生成者**: Codex（代码分析）
**用途**: 描述实现后的功能流程

**结构**:
```markdown

# 功能流程文档

## 功能 1：用户注册

### 流程图

[ASCII 流程图]

### 步骤详解

1. 用户输入邮箱和密码
2. 系统验证邮箱格式
3. 系统检查邮箱是否已注册
4. 系统生成验证码
5. 系统发送验证邮件
6. 用户点击验证链接
7. 系统创建用户账户
8. 系统返回登录令牌

### 错误处理

* 邮箱格式错误 → 返回 400

* 邮箱已注册 → 返回 409

* 验证超时 → 返回 408

### 性能指标

* 平均响应时间：200ms

* 99% 响应时间：500ms

* 吞吐量：1000 req/s

## 功能 2：用户登录

### 流程图

[ASCII 流程图]

### 步骤详解

1. 用户输入邮箱和密码
2. 系统查询用户
3. 系统验证密码
4. 系统生成 JWT Token
5. 系统返回 Token

### 错误处理

* 用户不存在 → 返回 401

* 密码错误 → 返回 401

* 账户被锁定 → 返回 403
```

```

  ↓

  ↓

  ↓
开发阶段
  ↓

  ↓

  ↓

  ↓

  ↓

```

## 文档版本管理

每个文档都包含版本信息：

```markdown
---
version: 1.0
created: 2026-03-08
updated: 2026-03-08
status: draft | review | approved | completed
---
```

版本状态说明：

* **draft**: 初稿，待审查

* **review**: 审查中

* **approved**: 已批准，可执行

* **completed**: 已完成，存档
